# 智能网络安全分析平台任务说明

## 任务目标

建设一个面向企业内部安全团队的智能网络安全分析平台。平台通过类 nanobot 的智能体框架接入 LLM，支持安全工程师使用自然语言下发任务，由 LLM 进行意图识别、任务规划和结果解释，再由后端受控任务编排器调用安全 CLI 工具完成钓鱼邮件分析、资产发现、存活探测、端口识别、漏洞扫描、弱口令扫描、日志分析和大屏展示。

本任务说明用于交给 Trellis 进行任务拆解和研发排期，不是 PRD。

## 已确认用户

首要用户：

- 企业内部安全团队的安全工程师。

辅助用户：

- 平台管理员：管理用户、角色、资产组白名单、工具配置、LLM Provider、全局策略和 kill switch。
- 安全审计员：查看任务、审计日志、工具参数、原始证据和报告，不发起扫描或修改配置。
- 管理层只读用户：查看大屏和汇总报表，不发起任务，不查看敏感原始输出。

明确不面向：

- 普通员工自助扫描。
- 外部客户自助使用。
- 多租户 MSSP 场景。
- 未登录或匿名公网用户。

## 核心入口

平台的主要入口是自然语言智能体对话框。

安全工程师输入自然语言指令后：

1. LLM 识别用户意图。
2. LLM 生成结构化任务计划。
3. 后端校验权限、资产组白名单、安全策略、工具参数 schema。
4. 后端任务编排器创建任务并调用对应 CLI 工具。
5. 平台结构化保存结果、保留证据链、生成摘要和报告。

支持 YOLO 模式：

- YOLO 模式开启后，在授权资产范围内可跳过人工二次确认直接执行。
- YOLO 不跳过权限校验、资产白名单校验、工具参数校验、速率限制、超时限制、审计记录。
- 如果 LLM 意图识别不确定，YOLO 不能猜测执行，应在对话栏展示待执行操作，用户一次点击确认后继续。

## 资产授权模型

平台使用资产组白名单控制执行范围。

白名单类型：

- 域名白名单，例如 `example.com`，可配置是否允许子域发现。
- IP / CIDR 白名单，例如 `10.0.0.0/24` 或 `192.168.1.10`。
- 资产组白名单，例如“办公网段”“生产外网”“测试环境”。

权限绑定：

- 白名单按资产组绑定到用户或角色。
- 安全工程师只能在自己有权限的资产组内发起任务。
- 管理员负责创建、修改、删除资产组白名单。
- 每次任务执行前都必须重新做目标归属校验，不能只依赖 LLM 规划阶段校验。
- 子域发现结果应回写为已发现资产或待确认资产，不能自动扩大到未授权根域。
- 白名单变更必须记录审计日志。

## LLM 能力边界

LLM 允许：

- 自然语言意图识别。
- 生成结构化任务计划。
- 解释扫描结果。
- 合并重复漏洞。
- 生成风险摘要、处置建议和报告草稿。
- 分析钓鱼邮件语义、诱导话术和 IOC。
- 基于脱敏后的日志事件和聚合指标解释攻击趋势。
- 解释工具失败原因并建议下一步。
- 在需要澄清时生成明确问题。

LLM 不允许：

- 直接执行扫描工具，例如 `nmap`、`hydra`、`nuclei`。
- 绕过资产白名单、权限、速率限制或超时限制。
- 自行扩大扫描范围。
- 接触明文弱密码。
- 读取未授权任务结果。
- 自动生成或替换爆破字典。
- 编造未由证据支持的漏洞结论。
- 将敏感原始样本、凭据、内网资产数据发送给未经批准的外部 LLM Provider。
- 接触未经脱敏的原始防火墙日志、Web 日志、Cookie、Token、URL 敏感参数或账号字段。

## 受控命令执行器

为提升 LLM 工作能力，平台可配置 LLM 允许执行的辅助 shell 命令白名单，例如：

- `ls`
- `cd`
- `pwd`
- `base64`
- `file`
- `cat`

执行限制：

- 命令只能通过受控命令执行器运行，不能给 LLM 真实系统 shell。
- 每个任务有独立工作目录，例如 `/workspace/tasks/{task_id}`。
- 辅助命令只能作用于当前任务工作目录。
- 禁止访问平台源码、系统目录、其他用户任务目录、凭据目录。
- 命令应按 argv 执行，不允许拼接完整 shell 字符串。
- 默认禁止管道、分号、逻辑操作符、命令替换、重定向和通配符展开。
- 每条命令必须有超时、输出大小限制和审计日志。
- `cd` 应实现为受控 working directory 切换，不是持久 shell 会话。
- 命令输出进入任务上下文前应进行必要脱敏。

## 安全工具执行器

扫描类工具由后端任务编排器调用，不由 LLM 直接执行。

MVP 集成工具：

- `Subfinder`：子域名发现。
- `Httpx`：存活探测、标题、状态码、技术栈。
- `Nmap`：端口扫描和服务识别。
- `Nuclei`：漏洞扫描。
- `Hydra`：弱口令扫描。
- `emlAnalyzer`：邮件解析。
- `Magika`：文件类型识别。

执行要求：

- 每个任务隔离运行。
- 工具参数来自后端 schema，不接受任意 shell 字符串。
- 所有目标先通过资产组白名单校验。
- 所有输出进入任务工作目录和结构化结果表。
- 有并发、速率、超时、资源限制。
- 支持全局 kill switch，能停止所有运行中任务。
- 可配置工具强度档位，尤其是 `Hydra`、`Nuclei`、`Nmap`。
- 推荐支持 Docker / sandbox / runner 节点，避免主应用进程直接执行扫描工具。

## 固定安全工作流

MVP 以固定安全工作流为主，LLM 负责选择工作流和填参，不允许自由编排任意工具链。

### 1. 钓鱼邮件分析工作流

入口：

- 邮件网关直连模式。
- 可选手动上传 `.eml` / `.msg` / 粘贴邮件原文。

邮件网关模式：

- 平台作为串联标记型邮件网关。
- 邮件流向：外部邮件服务器 -> 平台网关 -> 企业真实邮箱服务器。
- 平台可配置邮箱转发域名或 IP。
- 平台自动接收全部入站邮件，验证和分析后再转发。
- 如果判断为钓鱼邮件，在邮件元数据中添加标签后转发。
- 默认不删除、不隔离、不阻断邮件。
- 分析服务不可用时 fail-open，邮件继续转发，并添加分析失败标签。
- 网关自身不可用时，应允许邮件系统走备用 MX 或备用转发路径。

建议标签：

- `X-Security-Phishing: suspected`
- `X-Security-Risk-Score: 85`
- `X-Security-Task-ID: <task_id>`
- `X-Security-Analysis: unavailable`

分析能力：

- SPF、DKIM、DMARC 验证。
- 发件域信誉和 Header 异常检查。
- URL、附件、正文、IOC 提取。
- `Magika` 识别附件和文件类型。
- `emlAnalyzer` 解析邮件头、正文、附件、URL、IOC。
- LLM 生成风险判断、诱导话术分析、处置建议。

边界：

- 附件和 URL 在 MVP 中只做静态分析。
- 不自动点击 URL。
- 不自动打开附件。
- LLM 不作为 SMTP 投递决策的唯一判断来源。

### 2. 资产发现工作流

输入：

- 授权根域名。
- 授权资产组。

流程：

1. 使用 `Subfinder` 发现子域。
2. 使用 `Httpx` 探测存活、标题、状态码、技术栈。
3. 将结果写入资产库。
4. 对超出白名单的结果标记为待确认，不自动纳入可扫描范围。

### 3. 端口与服务识别工作流

输入：

- 授权 IP。
- 授权域名。
- 授权资产组。

流程：

1. 使用 `Nmap` 执行端口和服务识别。
2. 结构化保存端口、协议、服务、版本、指纹。
3. 结果进入资产详情、漏洞扫描输入和大屏统计。

### 4. 漏洞扫描工作流

输入：

- 授权资产。
- 授权资产组。
- 已发现服务。

流程：

1. 使用 `Nuclei` 按预配置模板集扫描。
2. 保存模板 ID、风险等级、命中证据、请求响应摘要。
3. LLM 合并重复漏洞并生成风险解释。
4. 结果进入漏洞列表、大屏和报告。

### 5. 弱口令扫描工作流

输入：

- 授权资产。
- 服务类型。
- 预配置字典。

规则：

- 安全工程师可直接发起弱口令扫描，不需要单独权限。
- 安全工程师可在任务执行中查看爆破出的明文弱密码。
- 明文弱密码不落库。
- LLM 不能接触明文弱密码。
- 普通测试报告中的弱密码必须脱敏。
- 弱密码明文单独导出需要独立权限。
- 明文弱密码只允许在任务执行期间或任务完成后的短时间窗口内查看和导出。
- 过期后彻底不可恢复，只保留脱敏结果。
- 查看、复制、导出明文弱密码必须记录审计日志，日志不能包含明文密码。

### 6. 综合扫描工作流

示例指令：

- “检查资产组 A 的暴露风险。”

流程：

1. 资产发现。
2. 存活探测。
3. 端口与服务识别。
4. 漏洞扫描。
5. 结果汇总。
6. LLM 生成风险摘要和处置建议。

### 7. 日志自动分析工作流

目标：

- 自动接入和分析防火墙日志、Web 日志。
- 不依赖安全工程师手动上传或手动导入日志文件。
- 从日志中识别攻击趋势、攻击类型、来源、目标和处置动作，并进入大屏展示。

日志来源：

- 防火墙日志：优先支持 Syslog 接收，包括 TCP / UDP / TLS Syslog，可扩展 CEF、LEEF、厂商 key-value 和 JSON 格式。
- Web 日志：优先支持 Nginx / Apache / 反向代理 / WAF 的 access log、error log、JSON log 或 Syslog 转发。
- MVP 采用平台提供 Syslog / 日志接收端，并由防火墙、WAF、Nginx / Apache 主动转发日志的方式。
- 轻量采集 Agent 作为可选扩展点保留，不作为 MVP 必需项。
- 不把手动文件导入作为 MVP 主路径。

流程：

1. 平台配置日志源、日志类型、解析格式、所属资产组和保留周期。
2. 日志源持续推送或采集器持续上报日志。
3. 日志接收服务写入原始证据存储，并生成可追溯事件 ID。
4. 日志解析器归一化时间、来源 IP、目标 IP / 域名、协议、端口、动作、规则 ID、严重级别、URL、HTTP 方法、状态码、User-Agent 等字段。
5. 攻击识别规则对日志事件进行分类，例如扫描探测、暴力破解、SQL 注入、XSS、目录遍历、命令执行、敏感路径访问、异常错误激增、WAF / 防火墙规则命中。
6. 聚合服务按时间窗口、攻击类型、来源、目标资产、动作、严重级别生成趋势指标。
7. 大屏展示日志攻击趋势图、Top 攻击类型、Top 来源、Top 目标资产、阻断 / 放行 / 告警分布和 Web 攻击 URI / 状态码分布。
8. LLM 只能基于脱敏后的结构化事件或聚合结果生成解释和摘要，不应直接接触敏感原始日志。

边界：

- 日志分析不等同于 SIEM 深度关联分析。
- 不自动攻击溯源，不生成攻击者画像。
- 不因日志中出现新目标而自动扩大资产白名单或扫描范围。
- 不基于日志自动执行封禁、阻断、扫描或漏洞利用。
- 原始日志可能包含内网地址、账号、Cookie、Token、URL 参数等敏感信息，进入 LLM 上下文前必须脱敏。

### 日志分析数据契约

日志源配置应至少包含：

- `source_id`、日志类型、防火墙 / WAF / Web 服务产品类型、接入协议、解析格式、所属资产组、启停状态和保留周期。
- Syslog 接入应区分 TCP、UDP、TLS，并记录监听地址、端口、证书配置和来源 IP 允许列表。
- Web 日志接入应记录 Nginx / Apache / 反向代理 / WAF 的日志格式版本，避免解析器隐式猜测字段含义。
- 日志源创建、修改、停用、删除和解析格式变更必须进入审计日志。

原始日志证据应至少包含：

- `raw_log_ref`、`source_id`、接收时间、原始事件时间、原始大小、校验摘要、截断标记、解析状态和脱敏状态。
- 原始日志正文可以存入受控证据存储或对象存储引用，但不应直接进入 LLM 上下文。
- 原始日志查看、检索、导出必须校验权限并记录审计；导出默认只允许脱敏后的结构化事件和聚合指标。

归一化日志事件应至少包含：

- `event_id`、`raw_log_ref`、`source_id`、日志类型、事件时间、接收时间、来源 IP / 端口、目标 IP / 域名 / 端口、协议、动作、规则 ID / 规则名、严重级别。
- Web 事件应额外包含 HTTP 方法、URI 路径、状态码、User-Agent 摘要、请求大小和响应大小；Cookie、Token、账号、完整查询参数等敏感字段不得默认展示。
- 攻击分类字段应包含攻击类型、分类规则 ID、置信度和说明，例如扫描探测、暴力破解、SQL 注入、XSS、目录遍历、命令执行、敏感路径访问、异常 4xx / 5xx 激增。
- 目标资产字段应包含 `target_asset_id` 或未命中状态；未命中资产库或白名单时只作为日志事件展示，不自动纳入可扫描范围。

趋势聚合结果应至少包含：

- 时间窗口、日志类型、攻击类型、严重级别、来源 IP / 网段、目标资产、处置动作和事件数量。
- 大屏只消费聚合结果和脱敏后的结构化事件，不直接读取原始日志正文。
- 聚合统计应支持空窗口展示，避免大屏把无数据误解为采集失败；采集失败应通过日志源健康状态单独展示。

## 任务状态模型

任务状态至少包括：

- `SUCCESS`：所有计划步骤完成，结果已结构化入库。
- `PARTIAL_SUCCESS`：部分工具成功、部分失败，成功结果仍展示，失败步骤明确标注。
- `FAILED`：关键前置步骤失败，导致后续无法执行。
- `NEEDS_CLARIFICATION`：LLM 识别不确定或缺少关键字段。
- `BLOCKED`：权限、白名单或安全策略阻断。
- `CANCELLED`：用户或 kill switch 主动终止。

失败路径要求：

- 工具失败时保留 stdout、stderr、退出码、工具版本、执行参数。
- 网络超时或临时失败可做有限重试，例如 1-2 次。
- 部分成功任务应产出可用结果，但报告状态必须为部分成功。
- 部分成功结果要明确标注失败步骤和数据不完整。
- 不因失败自动扩大扫描范围。
- 不因失败自动提高攻击强度。
- 不因失败自动切换或扩大爆破字典。

## 报告与导出

报告能力：

- 支持任务报告导出。
- 支持资产风险报告导出。
- 支持钓鱼邮件分析报告导出。
- 支持漏洞扫描报告导出。
- 支持弱口令扫描报告导出。
- 支持日志攻击态势报告导出，默认只导出结构化事件和聚合指标，不导出敏感原始日志。

弱口令报告规则：

- 普通测试报告中弱密码必须脱敏。
- 明文弱密码不落库。
- 明文弱密码单独导出需要独立权限。
- 明文弱密码导出只能在执行期或短时间窗口内完成。
- 明文导出必须记录审计日志，但审计日志不能包含明文密码。

## 大屏展示

MVP 大屏展示 7 类指标：

1. 任务态势
   - 今日任务数。
   - 运行中任务数。
   - 成功、部分成功、失败、阻断任务数。
   - 平均耗时。

2. 资产态势
   - 授权资产组数量。
   - 发现资产数。
   - 存活资产数。
   - 新增资产。
   - 暴露端口。
   - Top 服务。

3. 漏洞态势
   - 高危 / 中危 / 低危漏洞数量。
   - Top 漏洞类型。
   - Top 风险资产。
   - Nuclei 模板命中趋势。

4. 弱口令态势
   - 弱口令资产数。
   - 服务类型分布。
   - 趋势。
   - 不展示明文密码。

5. 钓鱼邮件态势
   - 今日邮件量。
   - 疑似钓鱼量。
   - 风险评分分布。
   - Top 诱导类型。
   - Top URL / 域名。
   - 附件类型分布。

6. YOLO / 智能体态势
   - 自然语言任务数。
   - YOLO 直接执行数。
   - 澄清次数。
   - 被白名单阻断次数。

7. 日志攻击态势
   - 防火墙日志事件量。
   - Web 日志事件量。
   - 攻击趋势图，按时间窗口展示攻击事件数量和严重级别变化。
   - Top 攻击类型，例如扫描探测、暴力破解、SQL 注入、XSS、目录遍历、命令执行。
   - Top 来源 IP / 网段。
   - Top 目标资产。
   - 防火墙 / WAF 动作分布，例如阻断、放行、告警。
   - Web 攻击 URI、HTTP 方法和状态码分布。
   - 新增异常峰值和持续攻击趋势提示。

明确不做：

- 3D 地球攻击动画。
- 无业务价值的动态粒子效果。
- 复杂 SIEM 关联分析。
- 自动攻击者画像。
- 基于日志的自动封禁、自动反制或自动扩大扫描范围。

## 审计要求

所有关键动作必须可追溯：

- 用户原始自然语言指令。
- LLM 意图识别结果。
- LLM 生成的结构化任务计划。
- 用户是否开启 YOLO。
- 澄清和确认记录。
- 权限校验结果。
- 资产白名单命中结果。
- 工具名称、版本、参数、开始时间、结束时间。
- 任务 stdout、stderr、退出码。
- 任务状态变更。
- 报告导出记录。
- 明文弱密码查看、复制、导出记录。
- 日志源创建、修改、停用和删除记录。
- 日志解析规则、攻击分类规则和保留周期变更记录。
- 原始日志查看、检索和导出记录。
- 日志解析失败、丢弃、截断和脱敏处理记录。
- 阻断原因。
- kill switch 操作。

## MVP 范围

MVP 包含：

- 自然语言智能体入口。
- YOLO 模式。
- LLM 意图识别、结构化任务计划和结果解释。
- LLM 辅助命令白名单执行器。
- 资产组白名单和用户 / 角色绑定。
- 固定安全工作流。
- 邮件网关串联标记型钓鱼邮件分析。
- 防火墙日志和 Web 日志自动接入、解析、攻击分类和趋势聚合。
- `Subfinder`、`Httpx`、`Nmap`、`Nuclei`、`Hydra`、`emlAnalyzer`、`Magika` 集成。
- 任务状态、审计日志、原始证据、结构化结果。
- 大屏展示。
- 报告导出。
- 弱口令明文执行期展示、短窗口导出、不落库。
- 全局 kill switch。

MVP 明确不做：

- 自动漏洞利用。
- 横向移动。
- 自动修复生产系统。
- 任意工具链自由编排。
- 任意 shell 执行。
- 未授权公网目标扫描。
- 邮件删除或隔离。
- SIEM / SOAR 深度双向联动。
- 手动日志文件导入作为主要日志分析路径。
- 基于日志自动封禁、自动反制或自动发起扫描。
- 多租户 MSSP 模式。
- 攻击溯源画像。
- 复杂 3D 可视化大屏。

## 验收标准

验收以端到端场景为准，而不是以单个功能模块清单为准。

### 场景 1：自然语言任务创建

给定安全工程师已登录并拥有资产组 A 权限，当输入“扫描资产组 A 的暴露风险”时：

- 系统应识别为综合扫描任务。
- 系统应生成结构化任务计划。
- 系统应校验资产组白名单。
- YOLO 开启时应直接创建并执行任务。
- 任务结果应能在任务详情、大屏和报告中查看。

### 场景 2：权限与白名单阻断

给定用户没有资产组 B 权限，当输入“扫描资产组 B”时：

- 系统应拒绝执行。
- 任务状态应为 `BLOCKED`。
- 系统应记录阻断原因和审计日志。
- 不应调用任何扫描工具。

### 场景 3：YOLO 不确定意图澄清

给定 YOLO 已开启，当用户输入“看看公司外网有没有问题”且系统无法确定具体资产组时：

- 系统不得猜测执行。
- 系统应在对话栏展示待确认操作或要求选择资产组。
- 用户一次点击确认后才能继续。

### 场景 4：邮件网关钓鱼分析

给定平台作为串联标记型邮件网关接收入站邮件时：

- 系统应执行 SPF、DKIM、DMARC 和邮件内容分析。
- 系统应调用 `emlAnalyzer` 和 `Magika`。
- 疑似钓鱼邮件应添加安全 Header 后转发。
- 非钓鱼邮件应正常转发。
- 分析服务不可用时应 fail-open，邮件继续转发并添加分析失败标签。

### 场景 5：综合扫描工具链

给定用户对授权资产组发起综合扫描时：

- 系统应至少完成 `Subfinder -> Httpx -> Nmap -> Nuclei` 链路。
- 每一步结果应结构化入库。
- 原始证据应可追溯。
- 工具参数应来自受控 schema。
- LLM 不得直接执行扫描工具命令。

### 场景 6：弱口令扫描

给定安全工程师对授权资产发起弱口令扫描时：

- 系统应调用 `Hydra` 执行扫描。
- 明文弱密码可在任务执行期或短时间窗口内查看。
- 明文弱密码不得落库。
- LLM 不得接触明文弱密码。
- 普通报告中弱密码必须脱敏。
- 明文弱密码单独导出必须校验独立权限并记录审计。

### 场景 7：部分成功

给定综合扫描中 `Subfinder` 和 `Httpx` 成功但 `Nmap` 超时：

- 任务状态应为 `PARTIAL_SUCCESS`。
- 成功结果应展示。
- 失败步骤应标红或明确标注。
- 报告状态应显示部分成功。
- 系统应保留失败工具的 stderr、退出码、执行参数和时间。

### 场景 8：LLM 边界

给定 LLM 生成了扫描计划：

- 后端必须重新校验权限、白名单和参数 schema。
- LLM 只能通过结构化任务间接触发扫描工具。
- LLM 只能执行配置白名单内的辅助命令。
- 辅助命令只能在当前任务隔离目录运行。
- 不允许 LLM 拼接任意 shell 字符串执行。

### 场景 9：大屏真实数据更新

给定系统已有真实任务、资产、漏洞、弱口令、邮件分析和日志分析数据：

- 大屏应展示任务、资产、漏洞、弱口令、钓鱼邮件、智能体执行和日志攻击态势 7 类指标。
- 指标应来源于真实任务数据。
- 弱口令大屏不得展示明文密码。
- 日志攻击态势应来源于自动接入的防火墙日志和 Web 日志，不依赖手动导入。

### 场景 10：审计闭环

给定任意任务创建、执行、失败、导出、阻断或 kill switch 操作：

- 系统应能追溯操作者、时间、输入、计划、权限校验、工具调用、结果和状态变化。
- 明文弱密码相关审计不得包含明文密码。

### 场景 11：日志自动分析和攻击趋势大屏

给定平台已配置防火墙日志源和 Web 日志源，且日志源持续发送日志：

- 系统应自动接收日志，不要求安全工程师手动上传或导入日志文件。
- 系统应保留原始日志证据或可追溯引用，并结构化保存归一化事件。
- 系统应识别常见攻击类型和防火墙 / WAF 处置动作。
- 系统应按时间窗口聚合攻击趋势，并在大屏展示趋势图。
- 大屏应展示 Top 攻击类型、Top 来源、Top 目标资产、动作分布和 Web 攻击 URI / 状态码分布。
- 未命中资产白名单或资产库的日志目标不得自动纳入可扫描范围。
- LLM 生成日志摘要时不得接触未经脱敏的敏感原始日志。
